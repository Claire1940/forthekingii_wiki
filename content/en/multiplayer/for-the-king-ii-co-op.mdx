<Section icon={Map} title="How to Set Up Your Multiplayer Session">

Setting up a session for **for the king ii co op** requires the host to manage the adventure settings and lobby privacy. Unlike some modern drop-in titles, this game requires all players to be present in the lobby before the adventure begins or before a saved game is resumed. This ensures that character assignments are handled correctly before the map data loads.

Follow these steps to initiate a new game or resume a campaign with your team:

<Steps>
  <Step num="1" title="Initialize the Multiplayer Lobby">
    The host must select 'Multiplayer' from the main menu. Here, you can choose to host a 'New Adventure' or 'Load Game'. Set a room name and a password to prevent random players from occupying slots intended for your friends.
  </Step>
  <Step num="2" title="Select Region and Privacy">
    Ensure all players are searching in the same geographical region (e.g., North America, Europe). If your friends cannot see the room, verify that the host's NAT type is open and the game version matches across all devices.
  </Step>
  <Step num="3" title="Assign Characters and Roles">
    Once all players have joined the lobby, the host can assign specific characters to each player. In a two-player game, each person typically controls two characters. In a four-player game, everyone manages one hero.
  </Step>
  <Step num="4" title="Configure House Rules">
    Before starting, use the 'House Rules' menu to adjust the difficulty. You can modify Chaos speed, life pools, and starting loadout points to suit your group's skill level.
  </Step>
</Steps>

<Callout type="warning" title="Rejoining Rule">
Players cannot join a session that is already in progress. If a player disconnects, the host must save the game, return to the main menu, recreate the lobby, and have everyone rejoin before loading the save.

<Section icon={Shield} title="Cooperative Combat and Battle Grid Tactics">

Combat in **for the king ii co op** takes place on a 2x4 grid. Cooperation is essential here because positioning affects which allies you can protect and which enemies you can hit. Unlike the first game, the sequel's grid system introduces "Guard" mechanics and "Splash" damage that can punish a poorly positioned team.

Effective communication during a battle should focus on Focus spending and target prioritization. Since Focus is a limited resource, players should discuss who will use their Focus to land a killing blow or a crucial stun.

| Combat Mechanic | How it Works in Co-op | Best Practice |
| :--- | :--- | :--- |
| **Guard/Protect** | Frontline units can absorb hits for the backline. | Keep your Blacksmith directly in front of the Herbalist. |
| **AOE/Splash** | Attacks can hit adjacent tiles or entire rows. | Spread out if the enemy has powerful row-based attacks. |
| **Focus Sharing** | Focus is individual, but its use benefits the whole team. | Use Focus on high-percentage rolls to prevent damage spikes. |
| **Secondary Actions** | Items and movement don't end the turn. | Use a potion or move to a better tile before attacking. |
